# 📱 GitHub Actions 手动触发完整指南

## 🎯 如何手动触发工作流

### 步骤1：进入 Actions 页面
1. 打开你的 GitHub 仓库
2. 点击顶部导航栏的 **Actions** 标签
3. 在左侧栏找到 **宠物自媒体每日创作** 工作流

### 步骤2：点击 Run workflow
1. 在右侧找到蓝色的 **Run workflow** 按钮
2. 点击后会弹出一个下拉菜单

### 步骤3：配置运行参数（可选）

你会看到以下3个可选参数：

#### 📝 参数说明

| 参数名 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| **指定生成的宠物品种** | 手动指定今天要生成哪个品种<br>留空则自动选择最久未生成的 | 空（自动） | `柯基`<br>`金毛`<br>`布偶猫` |
| **跳过飞书推送** | 勾选后不会发送飞书消息<br>适合测试时使用 | 不勾选 | ✅ 勾选 = 跳过 |
| **强制重新生成** | 忽略30天限制，强制生成<br>即使该品种最近生成过也会重新生成 | 不勾选 | ✅ 勾选 = 强制 |

### 步骤4：运行

配置完参数后，点击绿色的 **Run workflow** 按钮启动。

## 🔧 常见使用场景

### 场景1：测试系统是否正常
```yaml
指定生成的宠物品种: 柯基
跳过飞书推送: ✅ 勾选
强制重新生成: ✅ 勾选
```
**说明**：生成柯基的内容，但不发送飞书，用于快速测试

### 场景2：紧急需要特定品种内容
```yaml
指定生成的宠物品种: 布偶猫
跳过飞书推送: ❌ 不勾选
强制重新生成: ✅ 勾选
```
**说明**：立即生成布偶猫内容并推送到飞书，无论之前是否生成过

### 场景3：正常补充内容
```yaml
指定生成的宠物品种: 金毛
跳过飞书推送: ❌ 不勾选
强制重新生成: ❌ 不勾选
```
**说明**：生成金毛内容，如果30天内生成过则会跳过

### 场景4：完全自动模式
```yaml
指定生成的宠物品种: （留空）
跳过飞书推送: ❌ 不勾选
强制重新生成: ❌ 不勾选
```
**说明**：让系统自动选择最久未生成的品种，等同于定时任务

## 📊 查看运行结果

### 方法1：查看 Actions 日志
1. 点击正在运行或已完成的工作流
2. 点击 **generate-content** 任务
3. 展开各个步骤查看详细日志

### 方法2：查看生成的文件
运行成功后，仓库会新增/更新以下文件：
- `sent_pets_history.json` - 生成历史记录
- `pet_database.json` - 宠物数据库
- `prompt_品种名_日期.txt` - 生成的Prompt文件

### 方法3：检查飞书消息
如果没有跳过飞书推送，检查你的飞书群是否收到消息。

## ⚠️ 常见问题

### Q1: 为什么看不到 "Run workflow" 按钮？
**可能原因**：
1. ✅ 检查你是否有仓库的写权限（Collaborator或Owner）
2. ✅ 确认工作流文件在正确位置：`.github/workflows/pet-content-generator.yml`
3. ✅ 确认文件内容包含 `workflow_dispatch:` 配置
4. ✅ 尝试刷新页面或重新进入 Actions

### Q2: 点击 Run workflow 后没有反应？
**解决方法**：
1. 检查浏览器控制台是否有错误
2. 尝试清除浏览器缓存
3. 换一个浏览器试试
4. 等待几秒后刷新页面

### Q3: 运行失败，显示红色 ❌
**排查步骤**：
1. 点击失败的工作流查看详细日志
2. 检查 Secrets 是否正确配置：
   - `MINIMAX_API_KEY`
   - `FEISHU_WEBHOOK`
3. 检查 API 密钥是否有效
4. 检查网络连接是否正常

### Q4: 指定的品种名写错了怎么办？
系统会自动将新品种添加到数据库中，所以：
- ✅ 正确：`柯基`、`金毛`、`布偶猫`
- ⚠️ 会被添加但可能不符合预期：`柯基犬`、`小金毛`
- 建议使用数据库中已有的标准名称

### Q5: 如何查看数据库中有哪些品种？
查看仓库中的 `pet_database.json` 文件，或者参考 `pet_database_template.json`

## 🚀 高级技巧

### 技巧1：批量生成多个品种
虽然一次只能指定一个品种，但你可以：
1. 手动触发多次，每次指定不同品种
2. 都勾选"跳过飞书推送"避免群消息轰炸
3. 最后统一整理生成的 `prompt_*.txt` 文件

### 技巧2：定期手动补充热门品种
每周手动触发一次热门品种（柯基、金毛、布偶猫等），
确保这些高流量品种的内容保持更新。

### 技巧3：测试 Prompt 优化效果
1. 修改代码中的 Prompt 模板
2. 手动触发生成同一品种（勾选强制重新生成）
3. 对比前后两次生成的内容
4. 评估优化效果

### 技巧4：紧急发布内容
当你急需某个品种的内容时：
```yaml
指定品种: 目标品种
跳过飞书: ❌ 不勾选
强制重生: ✅ 勾选
```
几分钟内就能收到飞书推送的完整 Prompt。

## 💡 小提示

1. **首次使用建议**：先用测试模式（跳过飞书推送）运行一次，确认系统正常
2. **品种名规范**：建议使用简短的标准名称，如"柯基"而不是"威尔士柯基犬"
3. **定时任务不影响手动触发**：即使设置了每天10点自动运行，你仍可随时手动触发
4. **并发限制**：GitHub Actions 同一时间只能运行一个工作流实例
5. **日志保存**：运行日志会保存90天，可随时回看
